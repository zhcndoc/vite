# Cloudflare 支持 Vite 的使命

_2026 年 6 月 4 日_

![Cloudflare Supports Vite's Mission Cover Image](/og-image-cloudflare-supports-vite.webp)

今天标志着 Vite 进入一个新阶段的开始。VoidZero 正在加入 Cloudflare。你可以在 [VoidZero 的公告](https://voidzero.dev/posts/voidzero-cloudflare) 中阅读 Evan You 撰写的完整故事。在 [Cloudflare 的博客文章](https://blog.cloudflare.com/voidzero-joins-cloudflare/) 中，他们解释了为何他们要投资 Vite 技术栈，而它如今已成为 Web 基础设施的一部分。Vite 的团队治理、使命和理念都不会改变：

- Vite 仍然是 MIT 许可证下的开源项目。
- Vite 仍然保持供应商无关，用 vite 构建的应用旨在可在任何地方运行。
- Vite 继续由 [Vite 团队](/team) 负责管理，其中包括受雇于不同组织的成员以及独立成员。
- [Vite 的 Open Collective](https://opencollective.com/vite) 资金由 Vite 团队管理，用于资助独立核心团队成员、依赖项以及生态系统中的其他关键项目。
- 受雇于 VoidZero 的 Vite 团队成员都将加入 Cloudflare，并继续从事 Vite 的工作。

正如 VoidZero 和 Cloudflare 的公告中所述，以下项目同样适用：[Vitest](https://vitest.dev/team.html)、[Rolldown](https://rolldown.rs/team)、[Oxc](https://oxc.rs/team.html) 和 [Vite+](https://viteplus.dev/team)。

## Cloudflare 的 Vite 生态系统开源基金

Cloudflare 致力于帮助 Vite 保持中立，并且还宣布设立一项面向 Vite 生态系统的 100 万美元开源基金。再加上 Vite 的 Open Collective（仍由 Vite 团队管理），这些资金将使我们能够进一步改进 Vite 及其生态系统，包括：

- 资助生态系统中更多受欢迎的插件和工具。
- 采用核心团队成员津贴，以支持独立团队成员。
- 与 Vite 技术栈的各个团队密切合作，包括 Rolldown、Oxc 和 Vite+。
- 与来自框架、插件和部署平台的生态系统维护者密切合作。
- 与标准组织协作，更快采用新的 Web 平台特性。
- 更快地审计并发布安全问题修复。

其中许多工作我们已经开始推进，这将有助于加快这一进程。
